Security

Read-only by design

Neraium is designed for observational analysis and operator decision support, not control-layer writes.

Architecture

Data access boundaries

  • Read-only connectors to existing telemetry/historian sources.
  • No automated control actions or writeback commands to operational systems.
  • Scoped source mapping and explicit access controls per environment.

Handling

Data governance in pilot mode

  • Pilot scope defines datasets, retention windows, and role permissions.
  • Operator-facing findings can be reviewed without granting plant control privileges.
  • Environment boundaries are documented before ingestion begins.
