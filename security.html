Security

Read-only by design

Neraium is designed for observational analysis and operator decision support, not control-layer writes.

Architecture

Data access boundaries

Read-only telemetry model

Connectors ingest observational data only. Neraium does not write into control systems.

Scoped access controls

Source mappings and role boundaries are defined per environment before pilot start.

Operator-first outputs

Findings are designed for review and human decision making, not autonomous control actions.

Deployment boundaries

Pilot scope documents approved data sources, retention windows, and review workflows.

Controls

Trust-center checklist

  • Data access approved by environment owner before ingestion.
  • No production write paths enabled during pilot.
  • Role-permission mapping reviewed with operator stakeholders.
  • Simulated benchmark results clearly separated from production claims.
