Overview

Neraium operates as a read-only signal interpretation layer across live telemetry environments. Persistent structural deviation is mapped for context, persistence, and governed operator review.

01

Interpretation Model

Live telemetry is ingested as a read-only stream, normalized per-sensor across a rolling context window, then compared across correlated signal clusters. Deviation persistence is tracked before any interpretation artifact is surfaced.

Python normalization.py 
# Per-sensor z-score normalization over rolling context window
def normalize_stream(signal: np.ndarray, window: int = 30) -> np.ndarray:
    normalized = np.zeros_like(signal, dtype=np.float32)
    for i in range(len(signal)):
        start   = max(0, i - window)
        segment = signal[start : i + 1]
        mu      = np.mean(segment)
        std     = np.std(segment) + 1e-8  # avoid division by zero
        normalized[i] = (signal[i] - mu) / std
    return normalized

# Deviation persistence: flag if z-score exceeds threshold
# for at least `min_cycles` consecutive observations
def persistence_map(
    z: np.ndarray,
    threshold: float = 2.1,
    min_cycles: int = 5
) -> np.ndarray:
    flags  = np.abs(z) > threshold
    result = np.zeros_like(flags, dtype=np.int8)
    count  = 0
    for i, f in enumerate(flags):
        count    = count + 1 if f else 0
        result[i] = 1 if count >= min_cycles else 0
    return result
Python correlation.py 
# Lag-adjusted cross-signal correlation across sensor cluster
def cross_correlate(
    signals: dict[str, np.ndarray],
    max_lag: int = 10,
    r_threshold: float = 0.72
) -> dict[tuple, float]:
    keys  = list(signals.keys())
    pairs = {}
    for i in range(len(keys)):
        for j in range(i + 1, len(keys)):
            a, b   = signals[keys[i]], signals[keys[j]]
            best_r, best_lag = 0.0, 0
            for lag in range(-max_lag, max_lag + 1):
                b_shifted = np.roll(b, lag)
                r = np.corrcoef(a, b_shifted)[0, 1]
                if abs(r) > abs(best_r):
                    best_r, best_lag = r, lag
            if abs(best_r) >= r_threshold:
                pairs[(keys[i], keys[j])] = {
                    "r":   round(best_r, 4),
                    "lag": best_lag
                }
    return pairs
Parameter Type Default Description
window int 30 Rolling normalization window in cycles
threshold float 2.1 Z-score magnitude required for deviation flag
min_cycles int 5 Minimum consecutive flags before persistence is recorded
max_lag int 10 Maximum cycle lag for cross-signal correlation search
r_threshold float 0.72 Minimum Pearson r for a correlated pair to be retained

Signal Integrity

Every interpretation artifact carries a complete provenance record: originating sensor identifiers, normalization window applied, cycle at which persistence was first recorded, and the correlated signal pairs that contributed to classification. No inference is made without a traceable input chain.

Lineage hash is computed over the concatenation of: dataset identifier, unit identifier, sensor list, onset cycle, normalization parameters, and ISO timestamp. It is appended to every artifact at generation time and cannot be modified post-issuance.
04

Governance

Neraium structures outputs for institutional review. Traceable artifacts preserve lineage, persistence, and context so governance remains explicit at the point of decision. All outputs are versioned and immutable post-generation.

Property Value Description
read_only true No write operations issued to source systems at any stage
actuation false System cannot trigger control events or modify setpoints
decision_authority operator All action decisions remain exclusively with human operators
lineage_preserved true Full input chain recorded per artifact via SHA-256 hash
artifact_mutability immutable Artifacts are write-once; no post-generation modification permitted
temporal_persistence full record Onset cycle, duration, and peak values stored with each artifact
05

Boundary

Neraium does not actuate, modify, override, or control infrastructure systems. Decision authority remains exclusively with operators. The following operations are outside system scope by design.

Operation In Scope Notes
Telemetry ingestion (read) Yes Read-only stream access only
Signal normalization Yes Internal computation; no output to source system
Deviation artifact generation Yes Structured record surfaced for operator review
Setpoint modification No Explicitly excluded from system capability
Control signal issuance No No interface to actuation layer exists
Automated remediation No All remediation decisions remain with operators
Alert routing / escalation Optional Passive notification only; no priority enforcement
The read-only constraint is enforced at the integration layer, not as a policy. Neraium establishes no write credentials to source infrastructure at deployment.
