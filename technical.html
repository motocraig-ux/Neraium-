Technical Architecture

Neraium is a read-only infrastructure intelligence layer designed to detect structural deviation across complex water systems before failure occurs.

The platform ingests existing telemetry streams and analyzes multivariate system behavior to classify deterioration patterns that traditional monitoring systems fail to detect.

System Overview

Telemetry is collected from existing operational infrastructure including pumps, cooling systems, treatment systems, and distribution networks. Neraium analyzes signal relationships across the system rather than relying on individual alarm thresholds.

Telemetry Ingestion

Native Infrastructure Data

Neraium ingests telemetry from existing sensors and infrastructure systems including SCADA, BAS, historians, and control system data exports.

No Control Integration

The platform never connects to control paths and does not modify operational systems. All telemetry is ingested as observational data.

System Level Context

Signals are analyzed within the context of the full infrastructure system rather than isolated sensor thresholds.

Deviation Detection

Instead of relying on simple alarm thresholds, Neraium analyzes persistent deviation across multiple correlated signals.

When system relationships drift from baseline behavior in a sustained pattern, the platform classifies the deviation as potential structural deterioration.

This enables operators to detect infrastructure degradation significantly earlier than reactive maintenance models.

Operational Boundary

Read Only

Neraium never writes to operational systems or modifies infrastructure settings.

No Actuation

The platform does not control pumps, valves, chemical dosing systems, or treatment processes.

Operator Authority

All operational decisions remain entirely with system operators and infrastructure owners.

Deployment Model

Neraium integrates with existing infrastructure environments with minimal operational disruption.

Telemetry is analyzed externally without altering system configuration or introducing new operational dependencies.

This architecture allows infrastructure operators to gain system-level visibility while preserving strict operational control boundaries.
